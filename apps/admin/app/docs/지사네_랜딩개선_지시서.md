# 지사네 랜딩페이지 개선 지시서 (Claude Code 반영용)

> **전제**: Claude Code가 이미 구현한 Owner/Partner 랜딩(DB 실시간 통계 + 카테고리 탐색 + 전문서비스 쇼케이스, 7개 섹션)을 **개선**하는 지시서다. 전면 재작성이 아니라 아래 6개 지점만 수정한다.
> **출처**: 별도 검토 세션의 제3자 관점 + 박이사·의뢰자 최종 판단.
> **원칙**: 실체를 정직하게 보여준다(show, not fake). 플랫폼 중립성 유지. 랜딩은 가입 유도지 구매 페이지가 아니다.

---

## 개선 요약 (6개 지점)

| # | 개선점 | 현재 | 변경 |
|---|---|---|---|
| 1 | 초기 숫자 | 실제 DB 카운트(초라함) | 가짜 숫자 금지 → 정직한 프레임 전환 + 네트워크 규모 명시 |
| 2 | 전문서비스 쇼케이스 | "엔터랩스 전용" | "전문 파트너 서비스"로 중립화 (엔터랩스는 그중 하나) |
| 3 | 가격 노출 | 첫 화면에 가격 표시 | 가치·설명 중심, 가격은 상세로 (무료만 CTA 강조) |
| 4 | 시니어 톤 | 양호 | 노인 배려 티 배제 — 전문가 예우 톤 점검 |
| 5 | 콜드스타트 | 0명 "모집 중" | 흐름형 표현 + 프레임 보강 |
| 6 | 상대편 증명 | 카테고리별 수 ✅ | 유지(잘 반영됨) |

## 1. 초기 숫자 — 가짜 숫자 금지, 정직한 프레임 (⚠️ 최우선)

### 문제
초기 DB 실제 카운트(시니어 ~120명, 기업 ~20곳)가 방문자에게 초라해 보인다. 그렇다고 **DB 값에 +1,000, +100을 더하는 것은 금지** — 표시광고법상 허위·과장이며, 매칭 시 즉시 들통나 신뢰(지사네 핵심 자산)가 붕괴된다. 박이사 원칙("실체를 show")과도 모순.

### 지시
**A. 핵심 수치 4칸 그리드의 "숫자 프레임"을 전환한다.**
- 사람 수(초라)를 전면에 두지 말고, **전문성의 폭**을 전면에:
  - Owner: `7개 분야` · `28개 전문영역` · `140+ 서비스` · `만족도 4.8` (사람 수 대신 서비스 다양성)
  - Partner: `20+ 참여기업` · `열린 의뢰 N건` · `수수료 0%` · `완료 N건`
- 실제 사람 수가 유의미해지면(수백 명+) 그때 "참여 시니어 N명"을 노출.

**B. "현재 참여자 수"로 부풀린 수를 표시하지 않는다.** 대신 네트워크 규모를 정직하게 별도 표기 가능:
- 예: "부울경 4,500+ 전문가 네트워크 기반" (사실이면 OK — '참여자'가 아니라 '네트워크')
- 예: "사전 등록 전문가 모집 중" (준비 단계임을 정직하게)

**C. 데이터 레이어(landing-stats.ts)는 그대로 실제 값 유지.** 조작하지 않는다. 표시 레이어(page.tsx)에서 "무엇을 전면에 둘지"만 조정.

### 구현 힌트
```ts
// landing-stats.ts: 실제 값 그대로. 추가 파생 지표만 계산.
export interface OwnerLandingStats {
  totalPartners: number        // 실제 값 유지
  totalCategories: number      // 28 — 이걸 전면에
  totalServices: number        // 140+ — 신규, 전면에
  totalMajorFields: number     // 7 — 전면에
  totalCompletedDeals: number
  avgSatisfaction: number
  networkSize?: number         // 4500 등 '네트워크 규모'(참여자와 구분 표기)
  categoryCounts: [...]
}
```
표시:
```
[7]      [28]        [140+]     [4.8]
분야     전문영역     서비스      만족도
```
→ 숫자가 모두 "크고 정직"하다. 사람 수 대신 전문성의 폭.

## 2. 전문서비스 쇼케이스 — 엔터랩스 전용 → 플랫폼 중립

### 문제
현재 "엔터랩스 전문 컨설팅"으로 특정 업체를 내세운다. 지사네는 **플랫폼**이고 엔터랩스는 **입점 공급자 중 하나**다. 특정 업체 홍보판이 되면 다른 제휴 파트너가 들어올 이유가 없고 중립성이 깨진다.

### 지시
**A. 섹션 제목 중립화**
- "엔터랩스 전문 컨설팅" ❌ → **"전문 컨설팅 서비스"** 또는 **"검증된 전문 서비스"** ✅
- Partner: "시니어 전문가 역량 강화 교육" → **"전문가 역량 강화 프로그램"**(제공사 중립)

**B. 개별 서비스 카드에 제공사 표기를 작게 추가**
- 각 카드 하단에 `제공: 엔터랩스` 형태의 작은 라벨. 나중에 `제공: OO파트너`도 자연히 들어갈 구조.
- 데이터 구조를 `provider` 필드로 일반화(service-catalog.ts).

**C. 구조를 "입점 파트너 서비스" 프레임으로**
- 초기엔 엔터랩스가 대부분 채우되, *다른 업체가 들어올 그릇*으로 설계. 하드코딩 "엔터랩스" 제거, provider 기반 렌더링.

### 구현 힌트
```ts
// service-catalog.ts
interface ProService {
  id: string
  title: string
  valueDesc: string       // 가격 대신 '가치 설명' (개선점 3)
  duration?: string       // '1주' 등은 OK
  provider: string        // '엔터랩스' | 향후 다른 파트너
  isFree?: boolean        // 무료 여부(CTA 강조용)
  detailUrl: string       // 가격·상세는 이 페이지에서
}
```

## 3. 가격 노출 — 첫 화면 가격 제거, 가치 중심

### 문제
쇼케이스 카드에 "200,000원", "300,000원"이 첫 화면에 노출된다. 랜딩 첫 방문자는 "알아보는 단계"인데 가격부터 보면 거래처럼 느껴져 부담. 무료·유료가 섞여 혼란.

### 지시
**A. 카드 첫 화면에서 가격 제거 → 가치 설명으로 교체**
- "AI 사업 진단 · 200,000원" ❌ → **"AI 사업 진단 — 데이터 기반 의사결정 진단"** ✅
- 소요 기간("1주")은 유지 OK. **금액만 상세 페이지로 이동.**

**B. 무료 서비스는 CTA로 살림**
- "AX 무료 진단"은 진입장벽을 낮추니 첫 화면에 유지 — 단 **유료 가격과 시각적으로 분리**하고 "무료로 시작" 버튼으로.
- 무료는 강조(배지), 유료는 "자세히 보기"로 상세 유도.

**C. 카드 구성 (개선 후)**
```
┌──────────────────────────────────┐
│ ★ AX 진단              [무료]     │   ← 무료만 배지
│ AX 5대 적용축 진단 · 1주           │   ← 가치+기간, 가격 X
│ [무료로 시작하기]                  │
├──────────────────────────────────┤
│ AI 사업 진단                       │   ← 가격 없음
│ 데이터 기반 의사결정 진단 · 3일     │
│ 제공: 엔터랩스     [자세히 보기]    │   ← 가격은 상세로
└──────────────────────────────────┘
```

## 4. 시니어(Partner) 톤 — 노인 배려 티 배제

### 문제
시니어는 "배려받는 노인"이 아니라 "현역 전문가"로 대우받길 원한다. "은퇴 후에도", "디지털 케어", 글자 확대 광고 등은 노인 취급 낙인이 되어 거부감.

### 지시
**A. 카피 톤 점검 (은퇴/돌봄 프레임 → 전문가 예우/무장 프레임)**
- "은퇴 후에도 기다리는 기업이" → "20년+ 실력을 기다리는 기업이"(은퇴 언급 최소화)
- 현재 hero "경험으로 일하고, 정당한 대가를 받으세요"는 양호 — 유지.
- 교육 프로그램 톤: "낯설어도 괜찮습니다"(못함 전제) → "경험에 AI를 더합니다"(강화 프레임)

**B. 접근성은 유지하되 광고하지 않는다**
- 글자 크기는 처음부터 읽기 편한 적정 크기(광고 문구 없이). "시니어를 위해 크게 했어요" 같은 생색 금지.
- 전화/1:1 상담 옵션은 중립적 보조로 배치("1:1 상담으로 시작"), "전화로 간편 등록" 대문짝 지양.

## 5. 콜드스타트 — 흐름형 표현 보강

### 지시
- 카테고리 카드가 0명일 때 "모집 중"(현재 설계) — 좋음, 유지.
- 저조한 절대수 대신 **흐름형** 표현 병행:
  - "이번 주 새 의뢰 N건", "지금 전문가 모집 중인 분야" 등.
- 카테고리 탭에서 참여/의뢰가 있는 분야를 우선 노출, 0인 분야는 후순위.

## 6. 상대편 증명 — 유지 (잘 반영됨)

- Owner에 시니어 풀(카테고리별 수), Partner에 기업 의뢰 현황을 보여주는 현재 구조는 박이사 "상대편을 보고 결정" 원칙에 정확히 부합. **그대로 유지.**
- 단 숫자는 개선점 1·5 원칙(정직·흐름형) 적용.

## 검증 체크리스트 (개선 후)

- [ ] 핵심 수치에 **가짜로 부풀린 사람 수 없음** (DB 실제 값 + 전문성 폭 프레임)
- [ ] "네트워크 규모" 표기 시 '참여자'와 명확히 구분
- [ ] 전문서비스 섹션 제목·카드가 **특정 업체(엔터랩스) 전용 아님** (provider 기반)
- [ ] 쇼케이스 **첫 화면에 유료 가격 없음** (무료만 CTA, 유료는 상세로)
- [ ] 시니어 랜딩에 **노인 배려 생색 문구 없음** (전문가 예우 톤)
- [ ] 카테고리 0건은 "모집 중" + 흐름형 표현
- [ ] 상대편 증명(시니어 풀 / 기업 의뢰) 유지

## 핵심 원칙 3줄 요약
1. **정직**: 없는 숫자를 만들지 않는다. 사람 수가 적으면 전문성의 폭·네트워크 규모로 정직하게 채운다.
2. **중립**: 지사네는 플랫폼. 특정 업체 전용이 아니라 여러 전문 파트너가 들어올 그릇으로.
3. **절제**: 랜딩은 가입 유도. 가격·노인배려는 첫 화면에서 빼고, 가치와 예우를 보여준다.
